# 微信公众号格式约束与 Prompt 收紧设计

**目标**

让 ARK 输出的正文更贴近微信公众号可发布格式，继续使用“受控 Markdown -> 本地 HTML -> 微信草稿接口”的链路，不让 LLM 直接输出 HTML。

**结论**

- 微信草稿接口的正文最终接收 HTML，不是 Markdown
- 当前项目保留 Markdown 作为中间格式是合理的
- 但必须把 LLM 输出格式收紧到当前转换器可稳定支持的子集

**本次改动**

1. 收紧 `llm_writer.py` 的系统提示词
   - 明确只允许：
     - `#`
     - `##`
     - `###`
     - 普通段落
     - `-` 无序列表
     - `1.` 有序列表
     - 行内链接
     - `**加粗**`
   - 明确禁止：
     - 代码块
     - 表格
     - 引用块
     - 图片 Markdown
     - HTML 片段
     - 多层嵌套列表

2. 扩展 `markdown_to_html()`
   - 支持 `###`
   - 支持 `1.` 有序列表
   - 支持 `**加粗**`

3. 保持发布链路不变
   - LLM 输出 Markdown
   - 本地转 HTML
   - 微信 `draft/add`

**成功标准**

- Prompt 明确约束微信公众号安全格式
- 转换器能稳定处理更自然的小标题、编号列表和加粗
- 不引入 HTML 直写和复杂格式
- 全量测试通过
